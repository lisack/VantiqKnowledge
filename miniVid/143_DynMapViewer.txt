### Summary

The Dynamic Map Viewer widget in Vantiq Client allows developers to integrate Google Maps into their applications, leveraging Google Maps API features. This widget can display location data from data streams and is customizable through both simple properties and programmatic methods. The guide provides code examples for setting map properties, adding markers, drawing paths and regions, and using Google Maps services like Directions. It also highlights the potential need for a Google Cloud account for some services in the future.

The Dynamic Map Viewer widget The map widget in Vantiq Client is a Google Map.

Developers can therefore take advantage of Google Maps API features in their own user interfaces.

You'll find the dynamic map viewer widget in the Data Display section in the client's Ad tab.

The dynamic map view can take in location data from a data stream.

There are a couple of simple properties that can be set for the dynamic map viewer from the configuration pane, but most work with the map takes place programmatically.

Here we're accessing the map with two lines of code 1 calling the widget a map widget and then declaring A variable set to the dot map property of that map widget.

The code: 
```javascript
var mapWidget = client.getWidget("ParisMap");
var map = mapWidget.map;
```

After this, we can refer to map in the code and set further properties for it.

Let's start with the properties that determine the map display location and how closely to zoom in.

In this example, we're focused on the latitude and longitude for a point in the center of Paris and to the level where we can see the streets pretty easily.

The code:
```javascript
map.setCenter({lat: 41.85, lng: -87.65});
mapWidget.maptype = DynamicMapViewer.MAPTYPE_HYBRID;
map.setZoom(15);
```

Zoom settings range from 1 to 20, with 20 being the closest where individual buildings become visible.

Also, the type of map is easy to change by setting the map type property for the map widget.

The code:
```javascript
mapWidget.maptype = DynamicMapViewer.MAPTYPE_SATELLITE;
```

Putting spots and lines and regions on the dynamic map viewer is all well and good, but can we also take full advantage of some of the services that Google offers?

Well, that's a somewhat complicated answer.

There are services that still work, but they've been deprecated in favor of services that now require Google Cloud accounts with projects that can be billed for resource usage and so on.

That said, here's a free example with a dynamic Mac viewer using the legacy Directions service.

The code:
```javascript
const directionsService = new google.maps.DirectionsService();
const directionsRenderer = new google.maps.DirectionsRenderer();
var mapWidget = client.getWidget("ConUS");
var map = mapWidget.map;
map.setCenter({lat: 41.85, lng: -87.65});
mapWidget.maptype = DynamicMapViewer.MAPTYPE_HYBRID;
map.setZoom(15);

directionsRenderer.setMap(map);

const onChangeHandler = function () {
    calculateAndDisplayRoute(directionsService, directionsRenderer);
};

var dl1 = client.getWidget("start2");
dl1.addEventHandler("onChange",onChangeHandler);
var dl2 = client.getWidget("end2");
dl2.addEventHandler("onChange",onChangeHandler);

function calculateAndDisplayRoute(directionsService, directionsRenderer) {
  directionsService
    .route({
      origin: {
        query: client.getWidget("start2").boundValue
      },
      destination: {
        query: client.getWidget("end2").boundValue
      },
      travelMode: google.maps.TravelMode.DRIVING,
    })
    .then((response) => {
      directionsRenderer.setDirections(response);
    })
    .catch((e) => window.alert("Directions request failed due to " + e));
}
```

Here the start code is not only setting up the map, but also coding event handlers for any change done to the two drop down widgets also on the client page.

Each of these drop downs provides a list of string values of place names, and when one is changed, the function Calculate and Display route is invoking the Directions service to calculate the best route to connect the two places.

As mentioned earlier, this works for now without requiring a Google Cloud account, but there is no telling when Google will decide to retire this functionality for applications with basic map needs.

The Dynamic Map Viewer is an easy way to integrate a Google Map without having to supply a Google API key.
