The Vantiq Client By the end of this video you should know what the client is and how to move within it and how to set client properties.

The client builder is for building low code visual user interfaces.

Vantiq clients run on both desktop browsers and in the Vantiq mobile app, like the screen shown here.

The design strategy employed by the client builder is often described by the initials MVC for models, views and controllers.

Bottles refers to the data that the client ingests to be converted in some way for display in views which the developer can further operate using controllers like the JavaScript code in event handlers.

This is what the client builder looks like.

The AD side consists of widgets in a palette that can be dragged into the canvas, configured and used for client based application.

The Edit side is for setting client and individual page properties.

Events flow into clients via data streams.

Most often, data streams are connections to outbound service events, but can also be from sources or topics.

Many widget displays can be bound directly to data streams.

Developers can also write code to execute when data comes into the stream.

Data objects hold client variables at either the whole client scope or for individual client pages.

On the right you see the actions choices.

The plus side is for adding more data objects.

The trash can icon is for deleting them and the lightning button will build a form like layout based on the type.

Click on the three dot menu next to client to bring up the client properties menu.

Open it and you'll see five tabs.

Basic is where we can add custom JavaScript code and libraries.

All code within the client is JavaScript.

Still within the client properties, I want to highlight the Events, Theme, and Localization tabs.

The Events pane is where developers can add code to execute when the client experiences certain conditions.

Theme is for setting a consistent appearance scheme across all the pages.

Individual widgets can override these settings as needed.

Finally, Localization is for converting all text and labels in the client to other languages.

The last tab, Advanced, is where developers set the working characteristics of the client.

Let me start with the area in the menu that I circled.

Toggling on any of these choices will affect all the pages in the client.

Above the circled area is a section that determines how the client will run and who can see it.

Let me call your attention especially to the Mark as Launchable option.

When a client is launchable, it becomes a visible listing in either the Mobile Apps Start area or a browser's client launch window, so it can be run from either.

The three settings at the bottom are for using custom CSS in the client, being able to reuse what you've done in this client as a base for future clients, and whether or not to run the client if events don't come into the system for a prescribed length of time.

Clients can have any number of pages, the first of which is always called.

Start Pages have several events associated with them, such as Onstart, for which developers can write their own custom handlers.

To create a client, you can go to the Add menu and choose Client, then New Client.
