**Summary:**

The Semantic Search Service in Vantiq enhances AI capabilities by allowing it to understand specific business practices, policies, and expertise through semantic indexes. This service provides VAIL procedures to perform searches against a vector database, adding context for generative LLMs to craft responses. Key procedures include "answer question" and "answer question as sequence," which perform semantic searches to find relevant material before generating a response. Developers can adjust search configurations using parameters like min similarity and runtime config. The service also includes a similarity search procedure that returns relevant content rather than answers, and a Retrieve Chunks procedure for retrieving embeddings by ID. These tools make AI more informed and facilitate data access.

The Semantic Search Service Semantic indexes are what make your AI fully versed in your specific business practices, policies, and expertise.

The Semantic Search service provides VAIL procedures to access them.

The semantic service takes in a prompt or part of a conversation and performs searches against the vector database to add context for the generative LLM to use in its response.

The answer question and answer question as sequence are two of these procedures.

Given the name of the semantic index to search and the question to answer these procedures, perform a semantic search against the entries in the index to find material that best answers the question before having a generative LLM model, craft a full response based on those entries.

The min similarity and runtime config parameters allow the developer to change the default semantic index search configurations just for this procedure call as desired.

The QALLM parameter establishes which generative LLM the developer wants to craft the response or it will use the default.

Add a conversation ID if the answer should join a conversation.

Answer properties is a catch all for any other information the developer might want associated with the response.

The procedures return an object containing a string answer, its own metadata, and what the actual prompt was once the AI rephrased it to account for the search results.

The similarity search procedure doesn't return the answer, but rather the relevant content that the search found.

The required parameters are the same as the answer question procedure.

We still need the name of the index in the question to be answered, but the rest of the parameters have to do with how tightly the search will be controlled and how many text snippets will be returned.
