### Summary

This document provides an in-depth explanation of how embedding large language models (LLMs) function within Vantiq, particularly in relation to semantic searches. It describes the process of converting input into vectors, known as embeddings, which are stored in vector databases. The document outlines how these embeddings interact with semantic indexes to enhance the contextualization of prompts for generative LLMs. It also details the steps for creating and configuring semantic indexes and embedding LLMs in Vantiq, including the use of prebuilt configurations and the addition of documents to vector databases. Additionally, it explains the role of semantic indexes in providing context for generative AI responses and the configuration options available for developers.

Embedding large language models take in input and convert them to arrays of floating point numbers of a fixed size, called vectors.

Converted vectors are also called embeddings, and these are stored in vector databases.

The semantic index works in conjunction with both generative and embedding large language models and Vantiq's vector database in order to add context to prompts.

The embedding large language model takes the documentation that the developer enters into the semantic index, converts it into vectors, which are then stored in Vantiq's vector database.

When the generative large language model is given a prompt, the semantic index runs a search of the documentation in that database to add context for producing a more informed response.
