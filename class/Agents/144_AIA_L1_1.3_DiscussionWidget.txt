### Summary

The **Discussion Widget** in Vantiq is designed to facilitate human-AI interactions by tracking tasks and outcomes in multi-agent applications. It displays interactions when a user requests a task, which may require responses from multiple skills across different agents. The agent that first receives the task creates and owns the discussion, with all interactions becoming part of the discussion record. Discussions are distinct from conversations, which are single-threaded and involve user interactions with a service procedure calling a Large Language Model (LLM). The widget simplifies the complexity of managing discussions, allowing configuration to specify which agent handles the task. Agents can be configured to automatically generate, delete, and update discussions, but only the discussion owner can execute these procedures. The document also explains how toggling the discussion creation setting affects the availability of discussion-related procedures.

AI Discussions Discussions are designed for use in tracking tasks and outcomes in multi agent applications.

The Discussion widget displays human AI discussion interactions in the Vantiq client when a user makes a request that is a task.

Tasks might require responses from multiple skills in multiple agents to fully resolve, all of which are persisted as parts of a discussion.

The agent that first receives the task creates the discussion and owns it.

All individual agent interactions with the task become part of the discussion record in the agents state.

Conversations on the other hand, are for a user interacting with a service procedure that calls a LLM.

Conversations work as part of collaborations.

Discussions are part of task processing.

Conversations are single threaded.

Discussions may have multiple threads.

Discussion widgets automate a lot of complexity of using discussions.

Simply configure them to the agent that will handle the task.

The programmatic discussion widget properties and methods are largely concerned with discussion threads and the state of the discussion overall.

By default, any agent can own a discussion, but that can be configured to false or toggled off in the Agent service interface.

The JSON structure shown with creation disabled set to false is the default extension option.

Agents with discussion creation turned on also auto generate discussions, delete discussions, get by ID, discussions list, and discussions update status public procedures.

Only the discussion owner can run these procedures for their discussion.
