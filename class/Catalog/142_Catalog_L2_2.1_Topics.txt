**Summary:**
This content provides an in-depth explanation of how non-service events and topics function within the Vantiq platform. It begins by introducing the concept of catalogs and their role in managing events, particularly focusing on topics as conduits for event messages. The guide walks through creating a schema for an event type, setting up a topic, and scheduling events. It also covers how catalogs can broker event deliveries between namespaces and how to publish topics to a catalog. Additionally, it explains how to subscribe to events from different namespaces and how services can ingest these events. The content concludes by discussing the preference for service events over topics due to their encapsulation of resources and ease of application maintenance.

And now for the elephant in the room.

If topics are this easy to use, why are they so seldom mentioned in classes and examples?

Bantic discourages topic use in favor of service events.

Service events behave much the same way as topics from a pure transport angle.

Services can send events to each other, to clients, and to catalogs just as topics can.

But services encapsulate other like resources and the functionality associated with them for processing events and maintaining state for the application in memory.

From an application maintenance standpoint, seeing all related resources together in a service makes it much easier to follow the program's inner workings.
