As you saw in the demonstration, page templates are templates for client pages.

The main difference between client and page templates is the scope.

For instance, in this picture we see that the page has a sidebar, but that is because the client settings included a sidebar.

The page template is inserted into the client project as a page and therefore takes on whatever the client settings are, including the color scheme and navigation bars.

Data streams are a client scoped source, so page templates don't have them, but page templates can include page level data objects.

To create a page template, go to the context menu for that page and choose Save Page as Template, given a name and package name and save.

Selecting a page template for use starts from your client.

Choose Add Page from Template in the client context menu and choose the desired template from the list.

Both kinds of templates provide a starting point for further development.

All of the template code, data, widgets, etc.
