Command Line Interface Installation Vantiq's command line interface is intended to make it easier to do routine resource maintenance tasks in shell scripts.

In this video, we'll cover how to install the CLI and use it to connect to the Vantiq platform.

The Vantiq command line interface is a tool primarily for managing Vantiq resources from a terminal window, which makes it perfect for use in shell scripts.

At first glance, the CLI looks a little like just accessing the Vantiq API, and in fact the CLI can do a large subset of the things API queries can do.

But the CLI was created with Vantiq system resource management in mind, as we'll see in subsequent videos.

Here's what we mean by management tasks.

The Vantiq CLI interacts seamlessly with the local file system for importing and exporting individual resources or whole projects, performing bulk uploads of data, setting up the VCs server for version control, etc.

Installing the CLI is a straightforward process.

We'll go through each of these steps now.

To begin, remember that the CLI has one software dependency.

This product was written in Groovy, which is a form of Java, so there will need to be a Java installation on the host machine.

Go to oracle.com and look for their Java archives, where you'll find version 11, which is the version required.

Each platform has clear instructions on how to perform the download.

In Windows, it's an executable file.

On Macs, it's done with the standard installer, and so on.

Once you've installed Java, set the environment variable JAVA_HOME to the home directory for the installation.

Feel free to use this slide as a reference.

Next, download Vantiq CLI zip file from the Developer Resources pane shown before.

You may need to Scroll down a bit to find it, but that big button is hard to miss.

Expand the zip file wherever you'd like it to reside, and then update your PATH environment variable so that it can be accessed from anywhere on your system.

Technically, if you've gotten this far, you've successfully installed the Vantiq CLI, but there is one more job to do to make the product much easier to use.

In a hidden directory in your home directory called Vantiq, there will be a text file called Profile.

Edit this in in order to set up namespace access credentials.

HTTP client options can also be set in the profile.

That Oauth grouping does not have to have a particular name.

Here we see Oauth One as the name, but it could just as easily have been ABC.

Notice that in the profile we called Personal there's also a setting for the namespace name, which is especially helpful when connecting with a username and password.

The command to actually run the Vantiq CLI is Vantiq or Vantiq.bat in Windows, and then the command if no dash options are specified.

If you'd like to use another profile from the profile file, use the minus S option and specify by name which one you want.
