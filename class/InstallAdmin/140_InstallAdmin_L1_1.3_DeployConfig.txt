Deployment Configurations In this video, we're moving on to the deployment configuration step of the installation process.

After the setup cluster command gave us the two new YAML template files in the target cluster directory, we now have all the pieces we need to complete the tasks necessary before deploying Vantiqh itself.

In this video, we'll cover what those tasks are.

Here's a basic visualization of the components that make up a full Vantiq installation, spread across the Vantiq database and shared node pools that we saw in the infrastructure diagram earlier.

You can see the replication of the most vital pieces, as would be expected for a production system.

Be sure to unzip your Vantiq license in the target cluster slash deploy slash sensitive so you'll be found so it will be found by subsequent commands.

Now on to the deploy yaml file, which consists of overrides merged with helm charts from the Cates deploy repo with the most common properties listed and their defaults set or commented out.

Read the documentation carefully.

There are many variations based on your cloud provider and other circumstances, and long lists of the many properties that can be set along with the descriptions of what they are.

Here's just an example from the screenshot where we can override what we want for Influx DB, Grafonadv, etc.

The secrets YAML file is where access credentials to systems within the cluster will go.

Details of each of the line items are in the installation documentation, and also read the comments in the file itself.

In many cases, the credentials haven't been created yet, so by filling out the secrets YAML file, you're setting those up.

Once both the yaml files are modified, run the generate secrets command, which will store the sensitive information into the sensitive subdirectory as you see in the screenshot.

If you sealed the secrets, that information will be encrypted, otherwise it will be base 64.

Delete the secrets dot yaml file after a successful run of the generate secrets command in order to maintain security.

I'll double check that I'm in my deployment subdirectory and then run the generate secrets command.

This should only take a few seconds, after which we'll have several secrets files in our target cluster deploy sensitive subdirectory, which you just saw in the slide.

Setting all the configurations and making access points to the future installation is a fair bit of work the first time through, but in the next video we'll deploy the full Vantage server into our cluster and that's where the deploy dot yaml file will be most useful.
