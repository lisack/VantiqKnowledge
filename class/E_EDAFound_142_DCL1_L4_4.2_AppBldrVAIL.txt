Speaking of transformations, this is an activity pattern that can call a procedure in VAIL to be performed on every incoming event.

What the procedure returns is what the new event value will be.

If the transformation activity task will call a procedure to change events, is a procedure activity task the same thing by default?

The answer could be yes, but the returned value of a procedure activity task doesn't have to become the outbound event.

It can either become another property of the event, or the events aren't affected at all by what the procedure does.
