**Summary:**  
This video introduces the concept of components in Vantiq, which are custom modules that developers can create to streamline repetitive tasks in their projects. The video covers different types of components, including App Components, GenAI Components, and Client Components, explaining their purposes and how they can be created and used within Vantiq's builder tools. App Components are used in service visual event handlers, GenAI Components are custom tasks for GenAI procedures, and Client Components are custom widgets for clients. The video also provides a brief overview of where and how these components are created in the Vantiq IDE.

Each of the builder tools in Vantiq allow developers to create custom components.

Components are custom modules that developers can create to save themselves time and effort when building repetitive functionality in their projects.

Consumers don't see all the code and other work that went into them and can configure them for their own purposes based on what properties the developer chose to expose.

In this video, we'll do a brief survey of the types of components developers can make in Vantiq, starting with App Components.

These are custom made app activity tasks for use in service visual event handlers.

An app component can be as simple as a single VAIL task that performs a conversion that you, the programmer have to do often, like miles to kilometers.

App components can be composed of many other activity tasks too, as we'll see in our use case later.

The GenAI Builder also allows developers to create custom GenAI tasks.

Much like App components, GenAI components are essentially custom GenAI tasks with the details of their inner workings obscured.

But for the properties the developer has chosen to expose, GenAI components behave much like App components.

Just as an aside, if you're wondering how GenAI flow tasks in App Flows differ from GenAI custom components, it's a fair question.

A GenAI flow activity task, like a GenAI custom component, can be comprised of multiple pieces that collectively behave like 1 task.
